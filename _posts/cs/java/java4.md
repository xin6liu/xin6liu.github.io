# 多态

## 多态：对象的多种形态
1. 引用多态：
父类的引用指向本类/子类的对象
`Animal obj1 = new Animal();`
`Animal obj2 = new Dog();`
2. 方法多态
`obj1.eat();`
`obj2.eat();`

- 多态的**三个前提**：
1. 继承
2. 子类overwrite
3. 父类数据类型的引用指向子类的对象

- 多态成员**访问的特点**：
成员变量：编译运行都看左边（父类）
成员方法：编译看左边（父类），运行看右边（子类）
静态变量和方法：编译运行都看左边（父类）

总结：只有第二项--非静态的成员方法编译看左边，运行看右边。（动态绑定）

- 多态的**弊端**：不能使用子类特有的成员变量(属性)和特有的成员方法。

- 多态的**优点**：减少多余对象的创建，不用说为了使用子类的某个方法又去重新在堆内存中开辟一个新的子类对象。

## 引用类型转换

向上类型转换/自动类型提升:
`Dog d1 = new Dog();`
`Animal a1 = d1;`

向下类型转换/强制类型转换(需用instanceof):
`Dog d2 = a1;` //不行
`Dog d2 = (Dog)a1;` //可以
`Cat c1 = (Cat)a1;` // 不行，a1此时已经是DOG类型

通常写为：
```java
if (a1 instanceof Cat){
    Cat c1 = (Cat)a1;
}else{
    System.out.println("无法类型转换");
}
```
